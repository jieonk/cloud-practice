cURL Lab

이 서버는 cURL 실습을 위한 공개 엔드포인트를 제공합니다. 전역 레이트리밋: 분당 120회, /limited: 분당 20회. 기본 도메인: https://cloud-practice.onrender.com

OpenAPI 스펙

-1) 기초 실습 (예열)

cURL을 처음 써보는 분들을 위한 아주 기초 연습입니다.

# A) 단순 GET 요청 (HTML 받기)
curl https://cloud-practice.onrender.com

# B) 응답 헤더만 확인
curl -I https://cloud-practice.onrender.com

# C) 요청/응답 상세 보기 (-v)
curl -v https://cloud-practice.onrender.com

# D) 응답을 파일로 저장
curl -o index.html https://cloud-practice.onrender.com

0) 빠른 시작

curl -i "https://cloud-practice.onrender.com/echo?role=student" -H "X-Lab: curl"

1) 에코 (GET/POST)

보낸 메서드/헤더/쿼리/바디를 그대로 돌려줍니다.

# GET + 헤더
curl -i "https://cloud-practice.onrender.com/echo?x=1" -H "X-Lab: curl"

# POST (form)
curl -i -X POST https://cloud-practice.onrender.com/echo -d "hello=world&n=1"

# POST (JSON)
curl -i -X POST https://cloud-practice.onrender.com/echo \
  -H "Content-Type: application/json" \
  -d "{\"user\":\"alice\",\"score\":99}"

2) 기본 인증

기본 계정: admin/secret (환경변수로 변경 가능)

curl -i -u admin:secret https://cloud-practice.onrender.com/auth

3) 쿠키 실습 (필수 동의 쿠키)

lab_consent=true 쿠키가 없으면 403, 있으면 200.

# 3-1) 쿠키 없이 접근 → 403
curl -i https://cloud-practice.onrender.com/cookie/require

# 3-2) 서버가 쿠키 설정 → 200
curl -i "https://cloud-practice.onrender.com/cookie/set?value=true"
curl -i -b "lab_consent=true" https://cloud-practice.onrender.com/cookie/require

# 3-3) 쿠키 파일(jar) 저장/재사용
curl -c jar.txt "https://cloud-practice.onrender.com/cookie/set?value=true"
curl -i -b jar.txt https://cloud-practice.onrender.com/cookie/require

# 3-4) 쿠키 삭제
curl -i https://cloud-practice.onrender.com/cookie/clear
curl -i https://cloud-practice.onrender.com/cookie/require  # 다시 403

4) 파일 업로드(확장) – 폴더 선택 → 수정 → 업로드

/tmp/uploads 아래 안전한 영역만 사용합니다.

# A. 폴더 생성/선택
curl -X POST https://cloud-practice.onrender.com/fs/mkdir ^
  -H "Content-Type: application/json" ^
  -d "{\"dir\":\"classA\"}"
curl "https://cloud-practice.onrender.com/fs/list?dir=classA"

# B. 파일 수정(덮어쓰기)
curl -X POST https://cloud-practice.onrender.com/fs/mkdir -H "Content-Type: application/json" -d "{\"dir\":\"classA\"}"

curl "https://cloud-practice.onrender.com/fs/list?dir=classA"

echo "first version" > temp.txt

curl -F "file=@temp.txt" "https://cloud-practice.onrender.com/fs/upload?dir=classA"

curl -s "https://cloud-practice.onrender.com/fs/file?path=classA/temp.txt"


echo "second version" > temp2.txt

curl -X PUT --data-binary "@temp2.txt" "https://cloud-practice.onrender.com/fs/file?path=classA/temp.txt"
(curl -v -T temp2.txt "https://cloud-practice.onrender.com/fs/file?path=classA/temp.txt"
)

curl -s "https://cloud-practice.onrender.com/fs/file?path=classA/temp.txt"

# C. 여러 파일 업로드(반복)
for %f in (a.txt b.txt) do ^
  curl -F "file=@%f" "https://cloud-practice.onrender.com/fs/upload?dir=classA"

# D. 삭제
curl -X DELETE "https://cloud-practice.onrender.com/fs/file?path=classA/note.txt"

5) 지연 / 리다이렉트 / 상태코드

# 5-1) 1.5초 지연
curl -i https://cloud-practice.onrender.com/delay/1500

# 5-2) 리다이렉트 (따라가기: -L)
curl -i https://cloud-practice.onrender.com/redirect

curl -i -L https://cloud-practice.onrender.com/redirect

# 5-3) 임의 상태코드
curl -i https://cloud-practice.onrender.com/status/418

6) 레이트 리밋 체험

# Windows CMD
for /L %i in (1,1,25) do @curl -s -o NUL -w "%%{http_code}\n" ^
  https://cloud-practice.onrender.com/limited

# PowerShell
for ($i=1;$i -le 25;$i++) { curl -s -o $null -w "%{http_code}`n" `
  https://cloud-practice.onrender.com/limited }

# macOS/Linux (bash)
for i in {1..25}; do curl -s -o /dev/null -w "%{http_code}\n" \
  https://cloud-practice.onrender.com/limited; done

7) gzip / 페이지네이션

# gzip 응답 확인
# (1) 기본 호출: 자동 해제됨
curl https://cloud-practice.onrender.com/gzip

# (2) --compressed 옵션: 결과는 동일 (자동 해제된 JSON)
curl --compressed https://cloud-practice.onrender.com/gzip

# (3) --raw 옵션: 원본 gzip 바이트 출력 (터미널에선 깨져 보임)
curl --raw https://cloud-practice.onrender.com/gzip

# (4) 파일로 저장 후 직접 해제 (Linux/macOS)
curl -o out.gz https://cloud-practice.onrender.com/gzip
gunzip -c out.gz

# 페이지네이션
curl -s "https://cloud-practice.onrender.com/items?page=3&size=5"

8) 트러블슈팅 팁

# 상세 요청/응답 보기
curl -v https://cloud-practice.onrender.com/echo

# 헤더만 보기
curl -I https://cloud-practice.onrender.com/status/418

# 응답 저장
curl -o out.json "https://cloud-practice.onrender.com/echo?x=1"
